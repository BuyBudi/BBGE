## WHAT BBGE DOES TODAY

The extraction pipeline in artifacts/api-server/src/services/bbge/extractionPipeline.ts
orchestrates multiple extraction strategies in sequence: metadata → rendered_browser → 
ai_vision. Platform detection lives in platformDetector.ts and reads from 
config/bbge/platformConfigs.ts. All results are normalised into a NormalizedListing 
in normalizer.ts before returning.

The existing pipeline works well for eBay. Facebook Marketplace hits a login wall 
and falls back to AI vision with poor results. Gumtree, Depop, and Craigslist use 
the generic browser extractor with inconsistent results.

## WHAT TO ADD: APIFY AS A PIPELINE STRATEGY

Add Apify as a new extraction method called "apify" that slots into the existing 
pipeline alongside metadata, rendered_browser, and ai_vision. It should be tried 
BEFORE rendered_browser for platforms that have a known Apify actor, and used as 
a fallback AFTER rendered_browser for generic/unknown platforms.

## STEP 2 — New file: artifacts/api-server/src/services/bbge/apifyExtractor.ts

Create this file from scratch. It must:

- Import ApifyClient from "apify-client"
- Read APIFY_API_TOKEN from process.env — if missing, all functions return early 
  with a skipped:true result rather than throwing
- Export a function: isApifyConfigured(): boolean
- Export an interface ApifyExtractorResult with these fields (all nullable):
    title: string | null
    price: string | null
    description: string | null
    seller_name: string | null
    seller_profile_url: string | null
    seller_member_since: string | null
    seller_review_count: number | null
    seller_rating: number | null
    location: string | null
    condition: string | null
    category: string | null
    listed_date: string | null
    images: string[]
    attributes: Record<string, string>
    raw: Record<string, unknown>
    skipped: boolean
    error: string | null
    actor_used: string | null

- Export a map PLATFORM_ACTORS: Record<string, string> mapping platform names to 
  Apify actor IDs:
    facebook: "apify/facebook-marketplace-scraper"
    gumtree: "epctex/gumtree-scraper"
    depop: "barnett/depop-scraper"
    craigslist: "apify/craigslist-scraper"
  Do NOT include ebay here — eBay already works via its own API.

- Export the main function: 
  extractWithApify(url: string, platform: string): Promise<ApifyExtractorResult>

  This function must:
  1. Check isApifyConfigured() — return skipped:true result if not
  2. Look up actor ID from PLATFORM_ACTORS using platform name — if not found, 
     return skipped:true (do not use a generic fallback actor, only run for known platforms)
  3. Initialise ApifyClient with the token
  4. Call client.actor(actorId).call() with input: { startUrls: [{ url }], maxItems: 1 }
     and a 90 second timeout
  5. Fetch the dataset items from the completed run
  6. Take the first item and normalise it into ApifyExtractorResult using defensive 
     optional chaining — every field must fall back to null rather than throw
  7. The normalisation must handle field name variations across actors. Try these 
     fallback chains for each field:
       title: item.title ?? item.name ?? item.heading ?? null
       price: item.price ?? item.priceText ?? item.price_text ?? null
       description: item.description ?? item.body ?? item.details ?? null
       seller_name: item.sellerName ?? item.seller?.name ?? item.seller_name ?? null
       seller_profile_url: item.sellerUrl ?? item.seller?.profileUrl ?? null
       seller_member_since: item.memberSince ?? item.seller?.memberSince ?? null
       seller_review_count: item.reviewCount ?? item.seller?.reviewCount ?? null
       seller_rating: item.rating ?? item.seller?.rating ?? null
       location: item.location ?? item.locationText ?? item.city ?? null
       condition: item.condition ?? item.itemCondition ?? null
       category: item.category ?? item.breadcrumb ?? null
       listed_date: item.listedAt ?? item.postedAt ?? item.date ?? null
       images: item.images ?? item.imageUrls ?? item.photos ?? []
       attributes: item.attributes ?? item.specs ?? item.details_table ?? {}
  8. Return the normalised result with actor_used set to the actor ID, error: null
  9. Wrap everything in try/catch — on error return a result with error set to the 
     error message and all other fields null

     if (!isApifyConfigured()) {
       logger.info({ method }, "Apify skipped: APIFY_API_TOKEN not configured");
       const idx = methodsAttempted.lastIndexOf("apify");
       if (idx > -1) methodsAttempted[idx] = "apify_skipped";
       warnings.push("APIFY_API_TOKEN is not configured. Apify extraction skipped.");

         warnings.push(`Apify extraction error: ${apifyResult.error}`);
       } else if (!apifyResult.skipped && apifyResult.title) {

       }
